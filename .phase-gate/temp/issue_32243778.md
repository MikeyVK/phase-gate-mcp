# Bug: Copilot Chat disables MCP tools per session (misleading 'disabled by the user')

## Problem
In GitHub Copilot Chat met MCP tools worden tools soms uncallable mid-session met de misleidende melding 'Tool X is currently disabled by the user'. Een nieuwe chat-sessie herstelt de tool vaak. Root cause is onduidelijk.

## Expected Behavior

Identificatie van de categorie die 'disabled' veroorzaakt. Gedocumenteerde recovery flow. Minimaal één concrete fix (server-side hardening of VS Code workaround) die sessie-vergiftiging voorkomt.
## Actual Behavior

Server-side hardening volledig geïmplementeerd:
1. tool_error_handler (@mcp_server/core/error_handling.py) vangt alle MCPError-subklassen (ValidationError, PreflightError, ExecutionError, MCPSystemError, ConfigError) en converteert naar ToolResult.error() met behoud van error_code en file_path
2. Onverwachte exceptions worden via een BLE001-catch omgezet naar ToolResult.error('Unexpected error: ...') — geen unhandled exceptions die de tool session kunnen corrumperen
3. Logging verloopt uitsluitend via stderr (logging.StreamHandler(sys.stderr) in mcp_server/core/logging.py) — stdout-purity gegarandeerd
4. ToolResult-contract: consistent geïmplementeerd in alle tools
Root cause van 'disabled by the user' is niet geëlimeerd (VS Code client-side gedrag), maar alle server-side factoren die sessie-vergiftiging kunnen veroorzaken zijn aangepakt.
## Context

Standalone bug issue. Geen parent issue. De VS Code client-side oorzaak (tool disabling na failure) valt buiten scope van wat server-side opgelost kan worden. Acceptatiecriteria 1 (categorie identificeren) en 3 (concrete server-side fix) zijn voldaan.
