# Configureerbare MCP root directory: .st3 vervangen door instelbaar pad

## Problem
De directorynaam '.st3' is een projectspecifieke naam die is ontstaan tijdens de ontwikkeling van SimpleTraderV3. Deze naam is hardcoded op 50+ locaties door de gehele MCP server productiecode. Dit maakt de MCP server niet distribueerbaar als zelfstandig pakket: elke gebruiker krijgt een '.st3'-map, ook al heeft zijn project niets met SimpleTrader te maken. Dit blokkeert deployment als zelfstandige tool voor gebruik in andere projecten.

## Expected Behavior

De MCP server directory (nu '.st3') is configureerbaar via een instellingenbestand. Alle path-resolution verloopt via een centrale path-resolver op basis van workspace_root + config_root. Geen hardcoded '.st3' strings meer in productie- of testcode. Fallback: configureerbare directorynaam (bijv. '.mymcp'). Wheel-installatie plaatst standaard config-bestanden in de fallback-locatie.
## Actual Behavior

De directorynaam '.st3' is hardcoded op 50+ locaties in productiecode:
- mcp_server/managers/artifact_manager.py (regels 353, 574, 806, 814)
- mcp_server/core/phase_detection.py (regel 82): default arg Path('.st3/state.json')
- mcp_server/scaffolding/template_registry.py (regel 35): default arg Path('.st3/template_registry.json')
- mcp_server/managers/enforcement_runner.py (regel 30): display constant '.st3/config/enforcement.yaml'
- mcp_server/utils/template_config.py (regel 45): workspace_root = Path('.st3/templates')
- mcp_server/tools/admin_tools.py (regel 27): RESTART_MARKER_PATH = Path('.st3/.restart_marker')
- mcp_server/managers/phase_contract_resolver.py (regels 18-20): drie display path constants
- mcp_server/managers/phase_state_engine.py (regel 451): '.st3/state.json' in git porcelain command
- mcp_server/tools/project_tools.py (regels 279-280): string literals '.st3/deliverables.json' en '.st3/state.json'
Geen centraal pad-resolution mechanisme aanwezig. Deployment als zelfstandig pakket is momenteel niet mogelijk.
## Context

Voortgekomen uit finding F25 in issue #257 research (Config-First PSE architecture). Blokkeert deployment van de MCP server als zelfstandig, distribueerbaar pakket voor gebruik in andere projecten. Geen parent issue. #254 (Centralize .st3 config root via MCP setting) is een duplicate en wordt gesloten ten gunste van dit issue.
