# refactor(#271): make contracts.yaml SSOT for workflow-phase membership

Removes the `phases:` list from `workflows.yaml` and makes `contracts.yaml` the single authoritative source for workflow-phase membership, ordering, and exit gates. Eliminates a concrete DRY violation where two config files independently declared the same workflow-phase relationships.
## Changes
- `contracts.yaml`: new file — SSOT for all 6 workflows, phases, exit gates, merge policy, branch-local artifacts
- `phase_contracts.yaml`: deleted (replaced by `contracts.yaml`)
- `config_loader.py`: `load_contracts_config()` added; `load_phase_contracts_config()` removed
- `contracts_config.py`: new `ContractsConfig`, `WorkflowEntry`, `WorkflowPhaseEntry` models
- `workflows.yaml`: `phases:` list removed; file now contains only workflow metadata
- `WorkflowTemplate.phases` removed; validator inverted
- `PhaseStateEngine`, `ProjectManager`, `CreateIssueTool` migrated to `ContractsConfig` API
- `enforcement.yaml`: allow feature branches from `refactor/*` bases
- `project_tools.py`: `stdin=subprocess.DEVNULL` fixes Windows git reflog timeout (issue #307)
- Reference docs updated: `github.md`, `mcp_vision_reference.md`, `tools/project.md`

## Testing
6 TDD cycles with full test coverage. 16 schema tests (C1), 13 loader tests incl. 6-workflow roundtrip (C2), 4 rename tests (C3), 11 consumer migration tests (C4), blast-radius suite migration (C5), WorkflowTemplate.phases removal tests (C6). All existing tests pass.

## ⚠️ Breaking Changes

Any code or config that reads `workflows.yaml.phases[]` will no longer find that key. All internal consumers have been migrated. External consumers (if any) must migrate to `contracts.yaml`.
