# refactor(#283): Remove state.json contamination; add ready-phase enforcement

Fixes the structural root cause behind state.json/deliverables.json contaminating main. Replaces the unreliable .gitattributes merge=ours approach with (1) .gitignore for permanent exclusion and (2) a new EnforcementRunner pre-hook system that makes artifact contamination impossible by design: create_pr is gated on the ready phase, and git_add_or_commit auto-unstages branch-local artifacts in that phase.
## Changes
- Removed `.gitattributes` (merge=ours approach ‚Äî misleading and unreliable on GitHub server-side merges)
- Added `.st3/state.json`, `.st3/deliverables.json` and root ad-hoc `*.py` debug scripts to `.gitignore`
- Added `.st3/config/enforcement.yaml` ‚Äî declarative pre/post hook definitions per tool
- Added `.st3/config/phase_contracts.yaml` ‚Äî per-phase constraints (e.g. `ready` requires clean gate)
- Extended `workphases.yaml` with `ready` phase as final pre-PR gate
- Overhauled `EnforcementRunner` with pre-hook dispatch wired into tool invocation
- `create_pr` now blocked unless current workflow phase is `ready` (clear error + suggestion)
- `git_add_or_commit` in `ready` phase auto-unstages `.st3/state.json` and `.st3/deliverables.json` before commit
- Updated `docs/reference/mcp/tools/git.md` and `github.md` with enforcement notes
