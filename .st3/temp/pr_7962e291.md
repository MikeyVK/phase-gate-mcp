# refactor(#283): ready-phase enforcement — Model 1 branch-tip neutralization

Prevents state.json and deliverables.json from contaminating main by implementing a structural, automated enforcement system. Replaces the unreliable .gitattributes merge=ours strategy with a two-layer defence: (1) .gitignore for new checkouts, (2) branch-tip neutralization at commit-time for already-tracked paths.

The core mechanism: during a `ready`-phase `git_add_or_commit`, the `EnforcementRunner` emits `ExclusionNote` entries into `NoteContext`; `GitCommitTool` reads those notes and calls `GitAdapter.neutralize_to_base()` before committing — aligning excluded paths back to their merge-base state so they produce zero net delta.
## Changes
### Core infrastructure
- `mcp_server/core/operation_notes.py` — new module: typed note variants (`ExclusionNote`, `BlockerNote`, `RecoveryNote`, `SuggestionNote`, `CommitNote`), `NoteContext` container, `Renderable` protocol
- `mcp_server/core/exceptions.py` — removed `MCPError.hints`, `PreflightError(blockers=)`, `ExecutionError(recovery=)` (26 raise-sites migrated to typed notes)

### GitAdapter
- `commit()` gains `skip_paths: frozenset[str]` postcondition (C2)
- `neutralize_to_base(paths, base)` — new method; aligns each path to merge-base before caller commits (C7)

### GitManager
- `commit_with_scope()` gains `skip_paths` parameter
- Constructor accepts `WorkphasesConfig` for phase-aware validation

### EnforcementRunner
- Declarative rewrite: returns `None`, writes `ExclusionNote` to `NoteContext`; no git ops
- `default_base_branch: str` injected from `GitConfig` (C9)
- Phase-aware D3c messaging: neutralize-based, base-specific error messages

### GitCommitTool
- Terminal route: reads `ExclusionNote` → calls `neutralize_to_base` → commits with `skip_paths=frozenset()`
- 3-tier base resolution: `params.base` > `state.parent_branch` > `git_config.default_base_branch`

### server.py
- Threads `NoteContext` through `handle_call_tool`
- Wires `WorkphasesConfig` at 2 callsites; injects `default_base_branch` to `EnforcementRunner`

### Config
- `.st3/config/enforcement.yaml` — pre/post hooks for tools
- `.st3/config/phase_contracts.yaml` — per-phase constraints
- `.st3/config/workphases.yaml` — extended with `ready` phase

### Cleanup (C10)
- Removed `TestGitAdapterSkipPaths` mock-ordering class (§14 violation)
- Replaced stale C3/C6 integration tests with `test_model1_branch_tip_neutralization.py` (Scenario A/B/C)
- Removed ad-hoc debug scripts (`check_yaml.py`, `fix_yaml.py`, `revert_yaml.py`, `show_yaml.py`)
